Используемые уровни логирования:


Debug - логирование для дебага.

Information - логирование информации общего характера.

Warning - логирование предупреждений, потенциальных ошибок. Например, логирование сообщений, которые уже были приняты и обработаны.

Error - логирование ошибок системы.

Fatal - логирование наиболее критических ошибок, которые могут привести к частичному отказу системы.



Логирование взаимодействия с брокером сообщений Kafka

Шаблон при успешной отправке сообщения в брокер : "Сообщение {messageId} успешно опубликовано в топик {topic:l}, partition:{partition}, offset:{offSet}", где messageId - идентификатор сообщения, topic - наименование топика, partition - номер партишена, offset - номер оффсета.

Шаблон подписки на топик : "Подписка на топик {topicName}", где topicName - наименование топика.

Шаблон вычитывания сообщения из брокера : "Получено сообщение из топика {topicName}. Partition:{partition}, offset:{offset}, message:{message}, id:{messageId}, requestId:{requestId}", где topicName - наименование топика, partition - номер партишена, offset - номер оффсета, message- тела отправляемого соощения в json, messageId - идентификатор сообщения, requestId - идентификатор сессии.

Шаблон начала выполнения операции: "Начало выполнения операции: {requestName}. Входные параметры: {request}", где requestName - наименование команды, request- набо входных параметров.

Шаблон успешного окончания выполнения операции : "Операция {operation} выполнена успешно за {elapsedTime} мс." где operation - наименование команды, elapsedTime - время выполнения операции.

Шаблон неуспешного окончания выполнения операции: "Ошибка при выполнении операции {operation}", где operation- наименование команды.

Шаблон отклонения сообщения: "Отказ в обработке сообщения. Причина: {reason}", где reason- причина отказа
